## shell.open() → Command 方案（最终方案）

### v1（失败）
- **尝试**: `tauri.conf.json` 中 `"shell": {"open": {"scope": [{"name": "**"}]}}` 
- **结果**: Panic — `ShellAllowlistOpen` 反序列化失败

### v2（capabilities scope，未生效）
- **尝试**: `capabilities/default.json` 中 `shell:allow-open` 自定义 regex 添加 file:// 匹配
- **结果**: 错误依旧 — 旧二进制缓存未重建，或 Tauri2 shell.open scope 机制复杂

### v3（最终方案：Command 方式）
- **回退**: 用 `Command.create('explorer', ['/root,', dir]).execute()` (Windows) / `Command.create('open'/'xdg-open', [dir]).execute()` (macOS/Linux)
- **优点**: `shell:allow-execute` 已在 capabilities 中配置好 explorer/open/xdg-open，无需任何额外 scope 配置
- **tauri.conf.json**: 保持 `"shell": {"open": true}` (恢复原样)
- **capabilities/default.json**: 保留 shell:allow-open 自定义 regex（无副作用）

### v4: 打开路径改为根目录
- **需求**: 按钮应打开用户设置的 `baseDataDir` 或系统默认目录，不带项目 ID

## 修复：用户选择路径下图片加载 403

- **问题**: 用户通过文件对话框选择的图片（非 `$APPDATA` 路径）加载时报 403 Forbidden
- **根因**: `tauri.conf.json` 中 `assetProtocol.scope` 仅为 `["$APPDATA/**"]`，当 `baseDataDir` 被设置为自定义路径（如 `F:\新建文件夹\`），`copyFileToProjectData` 将文件拷贝到该目录后，`convertFileSrc` 生成的 `asset://localhost/...` URL 不在 scope 内，WebView 拒绝加载
- **修复**:
  - `tauri.conf.json`: `assetProtocol.scope` 从 `["$APPDATA/**"]` 改为 `["**"]` — 允许 asset 协议加载任意路径文件（桌面应用已有完整文件系统访问权限，无安全降级）
  - `store.clipboard.ts`: 图片粘贴路径新增 data URL 降级处理 — 当 `copyFileToProjectData` 返回 null 时，调用 `readFileToDataUrl` 读取为 base64 URL（匹配 `useNodeCreation.ts` 已有模式）
- **关联文件**: `src-tauri/tauri.conf.json`, `src/store/store.clipboard.ts`

## 上传文件对话框无文件可选修复

- **问题**: `uploadSourceFileToProject` 传入 `accept='*/*'`，Tauri 端将其转换为 `extensions: ['*/*']`，而 Tauri dialog filters 的 extensions 需要真正的文件扩展名（如 `png`、`mp3`），`'*/*'` 匹配不到任何文件，导致对话框为空
- **修复**: `fileService.ts` `uploadSourceFileToProject` 中新增判断 — 当 `accept` 为空或 `'*/*'` 时，传空 `filters: []` 让 Tauri 显示所有文件，而非错误地传 `extensions: ['*/*']`
- **关联文件**: `src/services/fileService.ts`
