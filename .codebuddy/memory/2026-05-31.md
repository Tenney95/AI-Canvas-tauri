## 修复：粘贴的视频文件不保存到项目数据目录

- **问题**：用户通过 Ctrl+V 或拖放粘贴视频文件时，视频没有保存到 `{appDataDir}/data/{projectId}/` 目录，而是以 base64 或原始文件路径引用
- **根因**：`copyFileToProjectData` 中 `MAX_MEDIA_FILE_SIZE = 100MB`，视频超过此大小即跳过拷贝，只返回原始路径的 `convertFileSrc`。同时 `kind: 'file'` 分支的 `saveBinaryToProjectData` 缺少 try-catch 和日志

- **修复**：
  1. **`fileService.ts`**：`MAX_MEDIA_FILE_SIZE` 从 100MB → 2GB，覆盖绝大多数视频文件
  2. **`useAppStore.ts` pasteExternalFromDataTransfer**：`kind: 'file'` 分支的保存块增加 try-catch + console.log/warn/error 日志，方便定位失败原因
- **影响范围**：`copyFileToProjectData` 调用处（`useNodeCreation.ts` Tauri 原生拖放、`useAppStore.ts` kind: 'file-path' 分支）一并受益

## 优化：拖拽/粘贴文件先创建节点再后台上传

- **需求**：拖拽和粘贴媒体文件时，先立即创建节点（显示加载动画），再在后台异步上传/拷贝文件，完成后更新节点状态
- **修改文件**：
  1. **`useAppStore.ts` pasteExternalFromDataTransfer**：
     - `kind: 'file'`（image/video/audio）：立即创建节点 `status: 'loading'`，IIFE 后台保存 Binary/DataUrl，成功 → `updateNodeData({ status: 'success', imageUrl/... })`，失败 → `updateNodeData({ status: 'error' })`
     - `kind: 'file-path'`（image/video/audio）：同样先创建 loading 节点，后台 `copyFileToProjectData` / `readFileToDataUrl` fallback
  2. **`useNodeCreation.ts` Tauri 原生拖放**（image/video/audio）：立即创建 loading 节点，`continue` 跳出主循环，`void (async () => {...})()` 后台拷贝并 `store.updateNodeData`
- **无需 UI 修改**：`ImageNode`/`VideoNode`/`AudioNode` 已有的 `status === 'loading'` 分支会渲染 spinner + 文案

## 新增：删除节点/项目时同步删除本地文件

- **需求**：删除节点时同时删除对应的本地文件；删除项目时删除整个项目数据文件夹
- **修改文件**：
  1. **`fileService.ts`**：
     - 新增 `deleteFile(filePath)` — Tauri 端调用 `remove` 删除单个文件，浏览器端 no-op
     - 新增 `removeDirRecursive(dirPath)` — 递归删除目录及其所有内容
     - 新增 `deleteProjectDataDir(projectId)` — 删除 `{appDataDir}/data/{projectId}/` 目录
     - 新增 `deleteNodeFile(nodeData)` — 快捷方法，从 nodeData 取 filePath 并调 `deleteFile`
     - 新增导入 `remove, readDir, DirEntry` from `@tauri-apps/plugin-fs`
  2. **`useAppStore.ts` deleteNode**：在删除节点前，查找节点 data 中的 `filePath` 并调用 `fileService.deleteNodeFile`（非阻塞）
  3. **`useAppStore.ts` deleteProject**：新增 `fileService.deleteProjectDataDir(id)` 调用（非阻塞）
  4. **`useKeyboardShortcuts.ts` Delete/Backspace**：该路径直接过滤节点，绕过 `deleteNode`，需额外添加对每个选中节点调 `fileService.deleteNodeFile`（非阻塞）
- **安全措施**：所有 `remove` 操作均有 try-catch，失败仅 console.warn，不阻塞 UI

  2. **`sidebar.css`**：新增 `.delete-confirm-overlay`、`.delete-confirm-dialog`、`.delete-confirm-icon`、`.delete-confirm-text`、`.delete-confirm-actions`、`.delete-confirm-btn` 全套样式
