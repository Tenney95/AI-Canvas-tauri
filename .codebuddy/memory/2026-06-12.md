- **修复新建节点默认模型报错「请先在底部模型选择器中选择一个模型」**：
  - 根因：所有新建 generator 节点的创建路径（`useNodeCreation` 四个 add 函数、`useCanvasContextMenu.addNodeAtCtxPos`）都没有给 `data.model` / `data.provider` 赋值，导致 `AINodeDialog.onSubmit` 校验失败
  - 修复：
    1. `useNodeCreation.ts` 新增 `loadDefaultModel(nodeType)` 工具函数，从 `localStorage` 的 `canvas-model-prefs` 键读取该节点类型上一次选择的模型，解析出 `model`（全值如 `'apimart/gpt-5.4'`）和 `provider`（如 `'apimart'`）
    2. `addTextNode` / `addImageNode`（generator 路径）/ `addVideoNode`（generator 路径）/ `addAudioNode`（generator 路径）：创建节点 data 时自动填充 `model` 和 `provider`
    3. `useCanvasContextMenu.ts`：同样新增 `loadDefaultModel` + `addNodeAtCtxPos` 中 generator 节点自动填充
  - 首次使用（无 localStorage 记录）时仍会提示选择模型，这是预期行为
- **项目切换后自动 Fit View**：
  - 根因：`ReactFlow.fitView` prop 只在组件首次挂载时执行一次，项目切换后仅更新 nodes/edges 数据不触发视口自适应。且已有的 F 键 `canvas-fit-view` 自定义事件无人监听（死功能）。
  - 修复（2 文件）：
    1. `Canvas.tsx`：在 `CanvasInner` 新增 `useEffect` 监听 `canvas-fit-view` 事件 → `requestAnimationFrame` 后调用 `reactFlowInstance.fitView({ padding: 0.2 })`
    2. `store.projects.ts`：4 处 `set()` 后通过 `setTimeout(..., 0)` 派发 `canvas-fit-view`：
       - `createProject`（新建空画布）
       - `switchProject`（切换有数据/空项目两条分支）
       - `deleteProject`（替换默认画布 / 加载下个项目数据两条分支）
  - 副作用：F 键 fit-view 快捷键现在起效了（之前事件派发但无人监听）
