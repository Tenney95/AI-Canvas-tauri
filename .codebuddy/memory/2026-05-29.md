### 修复遮罩画笔：alpha 叠加导致慢速绘制变不透明

**问题**：之前的 buffer + globalAlpha 方案无效，因为每次 `drawImage` 到主 canvas 仍用 `source-over`，alpha 照常叠加。慢速绘制时连续圆圈在重叠区域 alpha 累积至不透明。

### 修复遮罩画笔 alpha 累积（v3）：基线恢复 + 完整路径重绘

**根因**：前两次方案（buffer + globalAlpha、线段连接）都无法解决 alpha 累积问题，因为每一次 `stroke()` 都是独立的 `source-over` 操作，相邻绘制图元的重叠区域 alpha 必然叠加。

**唯一正确方案**：每帧恢复笔触前状态再一次性绘制完整路径。
- `pointerdown`：`getImageData` 保存基线 snapshot → 初始化 `mattingStrokePoints`
- `pointermove`：追加坐标点 → `putImageData(baseline)` 还原到笔触前 → 遍历所有点以 `moveTo + lineTo` 构建一个完整路径 → **一次 `stroke()` 完成绘制**
- 整个笔触作为一次连续的 stroke 操作，Canvas 内部不会对自相交路径产生 alpha 叠加 → 快慢画透明度完全一致

**改动**：
- 移除 `drawCircle` 和 `drawLineSegment` helper
- `mattingLastPoint` → `mattingStrokeBaseline`（ImageData）+ `mattingStrokePoints`（坐标数组）
- bucket 工具保持独立逻辑（一次性 flood fill）

- 修改文件：`nodes/ImageNode.tsx`
