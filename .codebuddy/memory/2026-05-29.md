### AudioNode 真实声纹 + 进度线（替代频谱动画）

**需求**：显示真实的音频声纹（非模拟正弦波），播放时不随音乐波动，而是在声纹上叠加进度线。

**修改文件**：`components/nodes/AudioNode.tsx`
- `decodeAndDrawWaveform()`：fetch → decodeAudioData → 逐像素提取峰值 → 绘制真实波形
  - 左声道 PCM 数据按 canvas 宽度分桶，每列取最大值
  - 使用 `createLinearGradient`（橙→亮橙→橙）纵向渐变着色
  - decode 失败时降级为水平细线
- `renderCanvas(canvas, data, progress)`：统一绘制入口，`progress<0` 时不画进度线
- `startProgressLoop()`：播放时 rAF 循环，实时计算 `currentTime/duration`，调用 `renderCanvas` 重绘
- 暂停状态：真实声纹 + ▶ 播放按钮覆盖层
- 播放状态：真实声纹 + 白色竖线进度线 + 左下角橙点闪烁时间标签
- 移除 AnalyserNode 频谱动画，AudioContext 仅用于 decode + `createMediaElementSource` → `destination`
- `waveformRef` 缓存解码后的 peaks，避免重复解码

**修改文件**：`styles/nodes.css`
- 移除 `.audio-playing-indicator` / `.audio-playing-bar` / `@keyframes audio-bar-pulse`
- 新增 `.audio-progress-bar-container`：左下角时间 + 闪烁状态点
- 新增 `@keyframes audio-dot-blink`：状态点呼吸闪烁
- 进度线直接在 Canvas 上绘制（`ctx.strokeStyle = '#e8e8ed'`），不依赖 CSS

### 修复 AudioNode createMediaElementSource 报错

**错误**：`InvalidStateError: HTMLMediaElement already connected previously to a different MediaElementSourceNode`

**根因**：AudioContext 关闭后，`<audio>` 元素上旧 `MediaElementSourceNode` 连接残留，再次 `createMediaElementSource` 时触发。

**修复**：完全移除 `createMediaElementSource`。AudioContext 仅用于 `decodeAudioData` 解码波形，`<audio>` 元素自身即可播放，无需通过 AudioContext 路由到 destination。

- 移除 `sourceCreatedRef`
- 移除 `setupAudio()` 函数
- `togglePlay` 直接 `audio.play()` / `audio.pause()`
