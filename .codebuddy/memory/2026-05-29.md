### API Key 测试连接功能完善

**需求**：根据 `/doc` 中的 API 文档实现各厂商的测试连接，有余额的显示余额，无余额的显示通过/失败，失败原因弹出气泡提示。

**新增文件**：`services/testConnection.ts`
- `testProviderConnection(provider, apiKey, baseUrl?)` → `TestResult`
- APIMart: POST `/chat/completions` ping → 检查 `choices`，无余额
- 火山方舟: GET `https://ark.cn-beijing.volces.com/ping` → 检查 `pong`，无余额
- RunningHUB (模型密钥): POST `accountStatus` → 返回 `remainCoins` 余额
- GRSAI: POST `/chat/completions` "你好" → 检查 `choices`，无余额
- OpenAI 兼容: POST `/chat/completions` ping → 检查 `choices`，无余额

**修改文件**：`components/SettingsPanel.tsx`
- `TestButton` 改版：接受 `state`/`onTest`，测试中显示 spinner + 禁用
- 新增 `ProviderStatusBadge`：成功=绿色"通过"，失败=红色"失败"+ hover 弹出错误气泡
- 新增 `ProviderBalanceBadge`：有余额时显示（如 RunningHUB 的"1100 积分"）
- `SettingsPanel` 新增 `testStates` 状态 + `handleTest` 处理函数
- 替换 GRSAI 硬编码的"通过"和"积分 4,967"为动态组件

**修改文件**：`styles/settings.css`
- 新增 `.settings-spin` 旋转动画
