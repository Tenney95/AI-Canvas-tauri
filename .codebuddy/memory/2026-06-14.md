# 2026-06-14

## ComfyUI 启动 — 最终方案

### 最终代码
- `spawn_new_console()`: `cmd /k program arg1 arg2` + `current_dir()` + `CREATE_NEW_CONSOLE`
  - `/k` 保活窗口：Python 崩溃也能看到错误
  - 程序路径和参数通过 Rust `Command::arg()` / `args()` 独立传递，Rust 只对含空格路径加引号，不破坏 cmd 解析
- `run_bat_script()`: `cmd /c bat_path` + `current_dir()` — bat 脚本由 ComfyUI 服务进程保持运行

### 踩过的坑
1. `cmd /K "cd /d \"...\" && \"...\""` — Rust args() 自动加外层引号 + 转义 → cmd 收到乱码 → 空命令
2. `cmd /K` + `raw_arg()` 直传 — arg 被整体当单个参数，cmd 不解析
3. `start "" /D "dir" "program" args` — Windows 找不到文件（start 双引号解析问题）
4. 直接 spawn `python.exe` — Python 报错崩溃，控制台立刻消失，看不到错误

### 当前状态
- `spawn_new_console`: `cmd /k "F:\...\python.exe" main.py --skip-manager-update`
  - `arg("/k")` + `raw_arg(&cmd_line)` — 命令拼成单个字符串，raw_arg 原样传递避免 Rust 自动引号破坏
- `run_bat_script`: `cmd /c run.bat`（从 bat 所在目录执行）
- 文件：`src-tauri/src/comfyui.rs`

### cmd /k 的坑
- `cmd /k` 语法是 `cmd /k <单个命令字符串>`，把命令拆成多个 args 会导致 cmd 只执行第一个 arg
- 必须用 `raw_arg()` 把完整命令字符串注入，因为 `arg()` 可能加外层引号破坏结构
