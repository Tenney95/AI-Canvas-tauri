## 外部链接打开支持（已修复 2 次）
- 点击"获取 Key"按钮现在通过 `@tauri-apps/plugin-shell` 的 `open()` 打开外部网页
- 新增依赖：Cargo `tauri-plugin-shell = "2"` + npm `@tauri-apps/plugin-shell: "^2"`
- `lib.rs` 注册 `tauri_plugin_shell::init()`
- `tauri.conf.json` 配置 `plugins.shell.open = true`
- `SettingsPanel.tsx` 的 `GetKeyButton` 使用 `<a>` 标签 + onClick：
  - 始终 preventDefault，然后尝试 Tauri shell open()
  - Tauri shell 不可用或失败时 fallback 到 `window.open()`
- **关键修复**：`capabilities/default.json` 必须添加 `shell:default` 和 `shell:allow-open` 权限，否则 Tauri 安全策略会拦截 open() 调用且无任何报错

## TextNode 浮动工具栏
- 为已有文本输出的选中 TextNode 添加浮动工具栏，出现在节点上方
- 三个按钮：复制（带 ✓ 反馈动画）、清除空行（3+ 连续空行→2 个空行）、全屏显示
- 复制功能使用 `navigator.clipboard.writeText` + `useCopyFeedback` hook（1.5s 视觉反馈）
- 全屏显示：fixed overlay 蒙层 + 居中面板，Escape 或点击蒙层关闭
- 工具栏仅在 `selected && hasOutput` 时渲染
- 按钮 click 均调用 `e.stopPropagation()` 防止 React Flow 拦截
- CSS：`nodes.css` 新增 `.text-toolbar`（bottom: calc(100%+6px) 定位在节点上方）、`.text-fullscreen-*` 系列样式

## TextNode 全屏编辑改造
- **删除**了 TextNode 的原有双击编辑功能（`isEditing`、`editValue`、`handleDoubleClick`、`handleEditSave`、`handleEditKeyDown`、`handleContentClick`）
- 全屏 overlay 中的 body 从只读 `<div>` 改为 `<textarea>`（`.text-fullscreen-editor`）
- 全屏编辑器通过 `onChange` 实时写回 `data.output`（`updateNodeData(id, { output: e.target.value })`）
- 打开全屏时自动聚焦 textarea 并将光标移到末尾
- 节点预览区域简化为纯 `<div>` 只读展示，去掉了 `data-tooltip="双击编辑"`

## TextNode 全屏 overlay — Portal 修复
- **问题**：全屏 overlay 渲染在 React Flow 节点的 DOM 树内部，被节点的 `overflow`/层叠上下文约束，尺寸被限制在节点范围内
- **修复**：使用 `createPortal(..., document.body)` 将全屏 overlay 传送到 body 下，完全脱离节点 DOM 约束
- `TextNode.tsx` 新增 `import { createPortal } from 'react-dom'`，
