## ComfyUI 工作流：只替换 @提及节点的值
- **问题**：调用 ComfyUI 工作流时，`injectPromptsIntoWorkflow`/`injectDimensionsIntoWorkflow`/`injectVideoParamsIntoWorkflow` 无差别遍历所有 IO 节点替换值，未 @ 的节点也被覆盖
- **修复**：
  - `injectPromptsIntoWorkflow`：从遍历所有 `ioNodeIds` 改为只遍历 `workflowInputs` 的 keys（被 @ 的节点）
  - `injectDimensionsIntoWorkflow`：新增 `mentionedNodeIds` 参数，有指定时只修改被 @ 的节点；无指定则全量注入（兼容旧行为）
  - `injectVideoParamsIntoWorkflow`：同上，新增 `mentionedNodeIds` 参数
  - 调用方 (`executeComfyUIGenerate`/`executeComfyUIVideoGenerate`) 传入 `workflowInputs ? Object.keys(workflowInputs) : undefined`
- 未 @ 的工作流节点保留原始 JSON 中的值不动

## ComfyUI Workflow 图片上传
- **需求**：@ 出 image 类型 IO 节点（LoadImage）时，将画布图片节点的图片上传到 ComfyUI 服务器，替换工作流 JSON 中 LoadImage 节点的 `inputs.image`
- **实现**（`aiService.ts`）：
  - `uploadImageToComfyUI(baseUrl, imageUrl)` — 支持 data URL 和远程 URL，上传到 `POST /upload/image`，返回 `{ name, subfolder, type }`
  - `injectImagesIntoWorkflow()` — 遍历 `workflowInputs`，找出 image 类型 IO 节点，resolve @ 引用得到图片 URL，upload，设置 `inputs.image = name` + `inputs.upload = 'image'`
  - `submitComfyUIWorkflow` 在 prompt 注入后调用 `await injectImagesIntoWorkflow()`
  - 新增 `import type { WorkflowIONode } from '../types'`
