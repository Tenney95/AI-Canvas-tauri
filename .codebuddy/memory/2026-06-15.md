## tailwind.config.js — Theme layer 改用 CSS 变量 → 回退为类覆盖方案 (13:00)

用户发现 `.text-canvas-text` 等 Tailwind 工具类在浅色主题下未生效。首次尝试将 `tailwind.config.js` token 改为 `var(--theme-*)`，但 Tailwind 构建时解析变量为 RGB 值（如 `rgb(232 232 237 / var(--tw-text-opacity))`），CSS 变量被固定编入产物，无法运行时切换。

- `.text-canvas-text` → `#1a1a2e`, `.text-canvas-text-secondary` → `#6b6b7b`, `.text-canvas-text-muted` → `#9a9aaa`
- `.bg-canvas-bg` → `#F5F0EB`, `.bg-canvas-surface` → `#FFFFFF`, `.bg-canvas-card` → `#F8F6F3`, `.bg-canvas-hover` → `#EDE8E0`
- `.border-canvas-border` → `#E5E0D8`, `.border-border-subtle` → `rgba(0,0,0,0.06)`, `.border-border-secondary` → `#d4d0c8`

## Header.tsx — 白色透明度适配 (13:05)

用户反馈 Header 背景在浅色主题下未适配。组件中多出使用 `white/[alpha]` 硬编码值，在浅色底上不可见或突兀。

### 改动
5 处 `white/[alpha]` 替换为 `canvas-*` token：
- `border-white/[0.06]` → `border-canvas-border`（外框）
- `border-white/[0.08]` → `border-canvas-border`（Logo 右侧分割线）
- `hover:bg-white/[0.05]` → `hover:bg-canvas-hover`（项目名悬浮）
- `bg-white/[0.08]` → `bg-canvas-hover`（中间竖线分割）
- `hover:bg-white/[0.08]` → `hover:bg-canvas-hover`（按钮悬浮）

这些 token 在 `base.css` 已有 `[data-theme='light']` 覆盖，切换浅色主题时自动适配。

## base.css — 追加透明度修饰符变体覆盖 (13:10)

用户发现 `text-canvas-text/90` 和 `text-canvas-text-secondary/80` 等带 Tailwind 透明度修饰符的类在浅色主题下未适配。根因：Tailwind 将 `/90` 编译为独立类（如 `.text-canvas-text\/90`），之前的 `.text-canvas-text` 覆盖无法命中。

- `.focus\:text-canvas-text\/90:focus` → `rgb(26 26 46 / 0.9)`（项目名输入框聚焦态）
- `.text-canvas-text-secondary\/80` → `rgb(107 107 123 / 0.8)`（项目名默认态）
