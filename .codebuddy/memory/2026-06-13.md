- **CropEditor.tsx: 修复 `preventDefault` passive 事件警告**：
  - React `onWheel` 合成事件默认 passive，不支持 `preventDefault()`
  - 改为 ref + `addEventListener('wheel', handler, { passive: false })` 显式非 passive 绑定
  - 移除 JSX 上的 `onWheel={handleWheel}`，通过 `useEffect` 挂载/卸载
- **crop.css: 修复图片溢出父容器问题**：
  - `.ReactCrop__child-wrapper>img` 之前的 `width: 100%; height: 100%` + `max-height: none; max-width: none` 覆盖了 react-image-crop 库的默认约束（`max-width: 100%; max-height: inherit`），导致图片按原始尺寸渲染，撑大容器链（crop-zoom-stage 7836px vs crop-stage 784px）
  - 改为 `max-width: 100%; max-height: 100%; width: auto; height: auto` — 自适应容器且保持原始比例
  - 同时给 `.crop-zoom-stage` 加上 `max-width: 100%; max-height: 100%` 防止缩放容器溢出
  - **回退（v3）**：仍然溢出 + 用户要求去掉缩放。彻底方案：
    - 移除 `crop-zoom-stage` 包裹层、`scale` 状态、滚轮缩放/双击重置全部代码
    - `ReactCrop` 直接放在 `crop-stage` 内
    - `handleConfirm` 中的 `scale` 计算简化为 `img.clientWidth`/`clientHeight`（scale 恒为 1）
    - CSS 核心修复：**所有层级只用 `max-height: 100%` 不设 `height: 100%`**，避免 `height: 100% → 图片原始尺寸 → 撑大容器` 的循环。删除了 `.crop-zoom-stage` 和 `.crop-zoom-indicator` 所有样式
  - **v4（最终）**：用户要求恢复缩放功能。关键修复：
    - `crop-zoom-stage` 改用 `display: inline-flex`（收缩包裹内容），**不设 `height: 100%`**，`max-width/max-height: 100%` 约束在 img 层级
    - `crop-stage` 保持 `overflow: hidden` + `position: relative`，缩放溢出由它裁剪
    - `transform: scale()` 不影响布局盒模型，layout 尺寸不变但视觉放大，超出部分被父级裁剪 — 这才是正确的 zoom 实现方式
    - 恢复 `stageRef`、`handleWheel`（passive:false）、`handleDoubleClick`、`scale` 状态、缩放指示器全部代码
