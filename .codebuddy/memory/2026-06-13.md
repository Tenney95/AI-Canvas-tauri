- **ComfyUI 启动 Bug 二次修复**（Windows 弹窗"找不到文件 'ComfyU'"）：
  - 根因 1：`start "ComfyUI"` — `start` 命令第一个引号参数为窗口标题，可能被某些 Windows 版本误解析为文件名，导致弹窗
  - 根因 2：`find_python()` 用 `Path::exists()` — 当 `python/` 目录存在时被误当作可执行文件返回，`start` 尝试"打开"目录
  - 修复：`start ""` 空标题（Windows 最可靠写法）+ `is_file()` 替代 `exists()` + 系统 Python 通过 `--version` 输出探测
- **ComfyUI 启动 Bug 三次修复**（Windows 弹窗"找不到\"\\'文件"）：
  - 根因：`cmd /c start ""` 在多级引号嵌套和路径反斜杠中极其脆弱，不同 Windows 版本解析行为不一致
  - 彻底方案：**放弃 `cmd /c start` 语法**，改用 Windows API `CREATE_NEW_CONSOLE` (0x00000010) 直接在新控制台窗口启动进程
    - `spawn_new_console()`：通过 `CommandExt::creation_flags(CREATE_NEW_CONSOLE)` 直接在新终端启动 `python.exe main.py`
    - `run_bat_script()`：用 `cmd /c <script> + creation_flags(CREATE_NEW_CONSOLE)` 在新终端运行 .bat 脚本
    - 移除 `run_in_terminal()`：不再需要 `cmd /c start` 的所有变体
    - `#[cfg(windows)]` 条件编译：`CommandExt`、`CREATE_NEW_CONSOLE` 仅在 Windows 编译；非 Windows 提供 fallback 函数
