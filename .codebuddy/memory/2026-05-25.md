## 修复: Tauri 环境快捷键失效
- **问题**: Tauri 2 webview 中 `Ctrl+S` 等快捷键被原生层先消费，`window.addEventListener('keydown')` 收不到事件
- **第一次修复** (无效): `document.addEventListener` + 捕获阶段 + `e.stopPropagation()` — JS 层面始终无法绕过 WebView2 浏览器级快捷键拦截
- **根本原因**: WebView2 在底层（ICoreWebView2Settings::AreBrowserAcceleratorKeysEnabled=true）消费了 Ctrl+S/Ctrl+Z 等浏览器快捷键，DOM 事件根本到不了 JavaScript
- **最终修复**: 引入 `tauri-plugin-global-shortcut`（v2）插件，在 Rust 层注册原生快捷键，绕过 webview
  - **依赖新增**: `package.json` → `@tauri-apps/plugin-global-shortcut: ^2`；`Cargo.toml` → `tauri-plugin-global-shortcut = "2"`
  - **Rust**: `src-tauri/src/lib.rs` → 注册 `tauri_plugin_global_shortcut::Builder::new().build()`
  - **权限**: `src-tauri/capabilities/default.json` → 新增 global-shortcut:default/allow-register/allow-unregister/allow-is-registered/allow-unregister-all
  - **前端**: `src/hooks/useKeyboardShortcuts.ts` → 双重策略：动态 import + register Ctrl+S/Alt+S/Ctrl+Z/Ctrl+Shift+Z/Ctrl+Y；同时保留 JS keydown 作为 Web 模式 fallback
  - **HMR 修复**: 新增 `active` 守卫防止卸载后回调触发；setup 前先 `unregisterAll()` 清除旧注册；cleanup 中 fire-and-forget 注销（异步但下次 mount 会先清）；解决 "Couldn't find callback id" 热重载警告

## 修复: 原生快捷键保存后 Toast 不显示 + 项目名称不同步
- **问题1**: 全局快捷键回调中用闭包 `saveCurrentProject`（无 `await`），原生 IPC 回调上下文可能导致异步链丢失，`showToast` 的 set() 未触发 React 渲染
  - **修复**: native 回调改为 `useAppStore.getState().saveCurrentProject()` 获取最新 store 引用
- **问题2**: `setProjectName` 只更新 `projectName` 字符串，侧边栏项目列表读取 `projects[].name`，改名后不保存则不同步
  - **修复**: `setProjectName` 改为同时更新 `projects` 数组中当前项目的 `name` 字段，即时生效
