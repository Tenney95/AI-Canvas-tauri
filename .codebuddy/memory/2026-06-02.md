# 2026-06-02

- **修复 / 指令直接触发时 prompt 未拼入{{ 文章内容 }}**: 问题根因是 `AINodeDialog.tsx` 的 `onSubmit` 通过闭包捕获 `data?.prompt`，`handleSlashSelect` 里 `onChange(filledPrompt)` 更新 store 后，`requestAnimationFrame` 回调里的 `onSubmit` 仍读取旧值。修复：`onSubmit` 改为 `useAppStore.getState().nodes` 同步读取最新 store 值；`PromptPanel.tsx` 移除不必要的 `requestAnimationFrame`。修改文件: `AINodeDialog.tsx`, `PromptPanel.tsx`
- **直接触发模式不再替换输入框内容**: `handleSlashSelect` 中 shouldTrigger=true 时：先 `onChange(filledPrompt)` 更新 store → `onSubmit()` 读取最新值 → `onChange(prompt)` 恢复输入框为原始文本。预设不会出现在输入框中，输入框保持用户原有文字。insert 模式保持不变。修改文件: `PromptPanel.tsx`
- **无 {{ 文章内容 }} 占位符时，用户输入拼到模板最上方**: `fillTemplate()` 若模板不含 `{{ 文章内容 }}`，则将 `currentPrompt` 拼接到模板最上方（双换行分隔），而不是丢失输入。修改文件: `slashCommands.ts`
