## generateImage.ts Provider Adapter 重构（已完成）
- **问题**：`generateImage.ts` 252 行内 5 个 `if (provider === ...)` 线性分支，`!response.ok → 解析 errorBody → throw` 错误处理样板重复 3 份；`generateText.ts` 内还有第 4 份。
- **方案**：抽 provider adapter + 共享 HTTP 工具，契合 AGENTS.md「平台化演进 / 厂商接入走 adapter」方向。
- **改动文件**（5 个）：

  - 新建 `src/services/ai/providers/standardImage.ts`（59 行）— 标准 OpenAI 兼容 `/images/generations` 流程，general 和 fallback 共用。
  - 新建 `src/services/ai/providers/volcengineImage.ts`（64 行）— Seedream 专属请求格式（normalizeSeedreamSize / image 字段 / watermark）。
  - 重构 `src/services/ai/generateImage.ts`（252→125 行）— switch 分流，每个 case 仅做 config 解析 + adapter 调用。
  - 重构 `src/services/ai/generateText.ts`（105→93 行）— 错误处理改用 `parseResponseError()`。
- **验证**：`tsc --noEmit` + `eslint` 均通过，0 error 0 warning。
- **收益**：错误处理样板 4 份 → 1 份（httpUtils.ts）；general 与 fallback 的重复请求逻辑合并为 `generateImageStandard()`；volcengine 独立隔离。
