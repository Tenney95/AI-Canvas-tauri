# AI Canvas 开发指南

> 本文档面向开发者，涵盖本地环境搭建、项目结构、开发约定、调试技巧和常见问题。
> 最后更新：2026-07-21

## 1. 环境准备

### 1.1 必需工具

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | `^20.19.0` 或 `>=22.12.0` | 满足 Vite 8 的运行要求 |
| npm | 随 Node.js 安装 | 包管理器 |
| Rust | stable | Tauri 编译工具链 |
| Visual Studio Build Tools | 2022 | Windows 上编译 Rust 需要 C++ 工具链 |

### 1.2 Windows 额外配置

安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)，选择「使用 C++ 的桌面开发」工作负载。

### 1.3 安装依赖

```bash
cd AI-Canvas-tauri
npm install
```

## 2. 启动开发

### 2.1 仅前端开发（不需要 Tauri 运行时）

```bash
npm run dev
```

启动后访问 `http://localhost:1420`。此模式下：

- 文件操作自动降级为浏览器 API（File Input + IndexedDB）
- 原生对话框不可用
- Tauri 插件功能受限
- 3D 导演台独立窗口和运行资源下载不可用

### 2.2 完整 Tauri 桌面应用开发

```bash
npm run tauri dev
```

此命令会：

1. 先启动 Vite 开发服务器（端口 1420）
2. 编译 Rust 后端
3. 打开 Tauri 桌面窗口

`npm run tauri dev` 不会拉取或启动 3D 导演台源码。首次创建导演台节点时，应用会按正式环境的流程下载固定构建产物。

### 2.3 生产构建

```bash
npm run build          # 构建前端 (dist/)
npm run tauri build    # 构建桌面安装包
```

### 2.4 3D 导演台开发模式

AI Canvas 在开发和生产环境中都通过 `director-desk://localhost/index.html` 加载已校验的固定构建产物。主项目不使用 iframe，不启动端口 `5178`，也不依赖 Git、Node.js 或 npm 来运行导演台。

需要修改导演台本体时，在独立仓库中开发：

```bash
git clone https://github.com/Tenney95/3d-director-desk.git
cd 3d-director-desk
npm install
npm run dev
```

独立仓库的 Vite 服务只用于导演台本体开发。AI Canvas 的集成验证仍应使用 GitHub Release 构建产物。

## 3. 项目约定

### 3.1 文件命名

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 组件文件 | PascalCase + `.tsx` | `TextNode.tsx` |
| Hook 文件 | `use` 前缀 + camelCase + `.ts` | `useNodeCreation.ts` |
| 服务文件 | camelCase + `Service.ts` | `fileService.ts` |
| Store 文件 | `store.` 前缀 + 职责 + `.ts` | `store.nodes.ts` |
| 样式文件 | kebab-case + `.css` | `model-selector.css` |
| 类型文件 | `index.ts` | `types/index.ts` |

### 3.2 导入顺序

```tsx
// 1. React / 框架
import { useState, useCallback } from "react";

// 2. 第三方库
import { useReactFlow } from "@xyflow/react";

// 3. 本地组件
import TextNode from "./nodes/TextNode";

// 4. Hooks
import { useAppStore } from "../store/useAppStore";

// 5. 类型
import type { BaseNodeData } from "../types";
```

### 3.3 组件编写规范

- 节点组件用 `React.memo` 包裹
- 接收 `{ data, selected }` props
- 数据变更通过 `useAppStore().updateNodeData(nodeId, data)`
- 禁止在节点组件内直接修改 store 状态

```tsx
// ✅ 正确
const TextNode = React.memo(({ data, selected }: NodeProps) => {
  const updateNodeData = useAppStore((s) => s.updateNodeData);
  // ...
});

// ❌ 错误
const TextNode = ({ data }: NodeProps) => {
  useAppStore.setState({ ... }); // 禁止
};
```

### 3.4 类型定义

所有全局类型定义在 `src/types/index.ts`：
- `NodeType` — 节点类型联合类型
- `BaseNodeData` — 节点数据接口
- `CanvasProject` — 项目元数据
- `AppConfig` / `ApiProviderConfig` — API 配置
- `ModelOption` / `ModelGroup` — 模型选项
- `WorkflowDefinition` — ComfyUI 工作流
- `UserPreset` — 用户预设
- `NodeGroup` — 节点分组

新增类型优先加到这里，禁止在组件文件中重复定义。

### 3.5 样式约定

- 业务样式优先使用 Tailwind class
- 禁止 `!important`、硬编码颜色值、内联 `style.cssText`
- 复用 `tailwind.config.js` 中的 `canvas-*` 颜色 token
- React Flow 样式覆盖统一放 `src/styles/reactflow.css`
- 新增组件独立样式放入 `src/styles/` 目录
- 节点类型 Header 使用语义色：文本=indigo、图像=green、视频=blue、音频=orange

### 3.6 状态管理约定

- 所有状态变更必须通过 Store Actions
- 不可变更新：使用扩展运算符或 `immer`-style 更新
- 新增 slice 在 `src/store/` 下新建 `store.xxx.ts`，在 `useAppStore.ts` 中聚合
- 组件内选择器保持精简，避免不必要的重渲染：

```tsx
// ✅ 精确选择
const nodes = useAppStore((s) => s.nodes);

// ❌ 全量选择（任何状态变更都会重渲染）
const store = useAppStore();
```

### 3.7 节点快捷键

画布数字快捷键由 `src/hooks/useKeyboardShortcuts.ts` 统一处理。不要在节点组件中新增全局 `keydown` 监听。

| 快捷键 | 节点 |
|---|---|
| `1` 到 `7` | 文本、图像、视频、音频、全景、动画、3D 导演台 |
| `Alt + 1` 到 `Alt + 5` | 文本、图像、视频、音频、Markdown 源节点 |

新增数字快捷键时，同步更新 `CanvasContextMenu.tsx` 和 `Sidebar.tsx` 的帮助说明。创建入口必须复用同一组节点尺寸、角色和初始状态。

## 5. 调试技巧

### 5.1 React DevTools

安装 React DevTools 浏览器扩展，开发模式下在浏览器控制台使用。

### 5.2 React Dev Inspector

开发模式下按 `Ctrl+Shift+C` 可以点击页面元素跳转到源码位置（需要 IDE 支持 `codebuddycn://` 协议）。

### 5.3 Tauri 调试

```bash
# 查看 Tauri 日志
npm run tauri dev

# Rust 后端日志会输出到终端
# 前端 console.log 也会同步到终端
```

### 5.4 IndexedDB 调试

在浏览器 DevTools → Application → IndexedDB → `ai-canvas-db` 查看持久化数据。

### 5.5 Zustand DevTools

可以在 `create()` 中启用 Redux DevTools 集成调试状态变更。

### 5.6 3D 导演台运行时调试

先确认当前固定版本和产物信息：

```bash
node -e "console.log(require('./scripts/director-desk-release.json'))"
```

在 Tauri 窗口中按 `7` 或通过画布右键菜单创建导演台节点。首次使用会显示下载确认，安装状态可在 **设置 > 存储健康 > 3D 导演台资源** 中查看。

运行以下定向检查：

```bash
npx vitest run tests/services/directorDeskRuntimeService.test.ts
npx vitest run tests/services/directorDeskWindowService.test.ts
cargo test --manifest-path src-tauri/Cargo.toml director_desk_runtime::tests --lib
```

浏览器模式没有 Tauri 本地协议和独立窗口，不能用于导演台集成验证。

## 6. 新增功能指南

### 6.1 新增节点类型

1. 在 `src/types/index.ts` 的 `NodeType` 中添加新类型
2. 创建 `src/components/nodes/NewTypeNode.tsx`
3. 在 `src/components/Canvas.tsx` 的 `nodeTypes` 映射中注册
4. 在 `src/components/Sidebar.tsx` 的节点列表中添加入口
5. 在 `src/hooks/useNodeCreation.ts` 中添加创建方法
6. 在 `src/services/aiService.ts` 中添加对应的生成适配器（如需要）

### 6.2 新增 AI 提供商

### 6.4 更新 3D 导演台版本

导演台版本由 `scripts/director-desk-release.json` 固定，不跟随分支或 GitHub `latest`。先在 `Tenney95/3d-director-desk` 发布带 tag 的构建产物，再更新主项目。

1. 确认 Release 包含 `director-desk-vX.Y.Z.tar.gz` 和对应的 `.sha256` 文件。
2. 在 AI Canvas 仓库运行版本同步命令：

```bash
npm run director:update -- v0.3.2
```

3. 检查 `scripts/director-desk-release.json` 中的版本、URL、SHA-256 和体积差异。
4. 运行同步脚本、前端和 Rust 定向测试：

```bash
npx vitest run tests/scripts/updateDirectorDeskRelease.test.mjs
npx vitest run tests/services/directorDeskRuntimeService.test.ts
npx vitest run tests/services/directorDeskWindowService.test.ts
cargo test --manifest-path src-tauri/Cargo.toml director_desk_runtime::tests --lib
cargo check --manifest-path src-tauri/Cargo.toml --lib
```

5. 运行 `npm run build` 和 `npm run tauri build`，再发布新版 AI Canvas。

同步脚本会流式下载 Release 包，校验 SHA-256，检查 tar 路径、文件类型、入口和协议元数据，并原子更新清单。Rust 使用 `include_str!` 编译该清单，因此只替换远端 Release 不会更新已发布的 AI Canvas。

## 7. 常见问题

### Q: 前端开发模式下文件操作不可用？
A: 正常现象。Tauri 插件（fs、dialog）仅在 Tauri 桌面环境中可用。浏览器模式下自动降级为 File Input API + IndexedDB。

### Q: 如何清除所有本地数据？
A: 打开浏览器 DevTools → Application → IndexedDB → 删除 `ai-canvas-db` 数据库。

### Q: Tailwind class 不生效？
A: 检查 `tailwind.config.js` 的 `content` 配置是否包含对应文件路径。新增文件后可能需要重启开发服务器。

### Q: Tauri 构建失败？
A: 常见原因：
- Rust 工具链未安装或版本过低
- Visual Studio Build Tools 未安装
- `src-tauri/target/` 缓存损坏：删除后重试

### Q: Cargo 无法删除 `ai-canvas-tauri.exe`？

A: 旧的 Tauri 窗口仍占用 `src-tauri/target/debug/ai-canvas-tauri.exe`。先关闭应用窗口，并在任务管理器中结束残留的 `ai-canvas-tauri.exe`，再运行 `npm run tauri dev`。不要在进程仍运行时删除整个 `target` 目录。

### Q: 3D 导演台反复提示下载？

A: 应用只识别固定清单中的版本。检查 `scripts/director-desk-release.json`，确认本地目录包含匹配版本的 `index.html` 和 `director-desk-release.json`。开发环境可以在 **设置 > 存储健康** 删除资源后重新下载。

### Q: 3D 导演台下载失败？

A: 检查 GitHub Release 是否公开、产物名是否与清单一致，以及 `.sha256` 是否绑定同一文件。应用还会拒绝超过限制、包含不安全路径或协议元数据不匹配的归档。
