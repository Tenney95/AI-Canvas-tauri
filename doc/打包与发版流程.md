# 打包与发版流程

> 适用版本：v0.2.7+（已启用 `createUpdaterArtifacts` 与自动更新）

## 前置准备

### 需要的文件 / 配置

| 文件 | 作用 |
|---|---|
| `package.json` → `version` | 前端版本号（唯一来源） |
| `src-tauri/tauri.conf.json` → `version` | Tauri 版本号，运行时读取 |
| `src-tauri/Cargo.toml` → `package.version` | Rust 编译版本号 |
| `src-tauri/update-key.key` | 签名私钥（**敏感，勿提交**） |
| `src-tauri/update-key.key.pub` | 签名公钥（已写入 `tauri.conf.json` 的 `plugins.updater.pubkey`） |
| `src-tauri/tauri.conf.json` → `bundle.createUpdaterArtifacts` | 设为 `true`，构建时生成 `.sig` 签名文件 |

### GitHub Secrets（无需每次修改）

在仓库 **Settings → Secrets and variables → Actions** 中配置：

| Secret | 值 |
|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | `update-key.key` 文件内容（一行 `dW50cnVzdGVk...`） |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 生成密钥时设置的密码（当前为 `x`） |

> 密钥通过 `npx tauri signer generate -w ./src-tauri/update-key.key -f -p "密码" --ci` 生成。
> 每次重新生成密钥后，必须连同 `tauri.conf.json` 中的 `pubkey` 一起更新。

## 第一步：版本号同步

### 1.1 修改 `package.json` 的 `version`

例如 `"0.2.6"` → `"0.2.7"`

### 1.2 运行同步脚本

```powershell
npm run sync-version
```

这会自动同步版本号到：
- `src-tauri/Cargo.toml`（Rust 编译时不再显示旧版本）
- `README.md`（版本 badge）

### 1.3 手动修改 `src-tauri/tauri.conf.json`

```json
"version": "0.2.7"
```

> 三处版本号（`package.json`、`tauri.conf.json`、`Cargo.toml`）必须一致。

---

## 第二步：本地构建

### 2.1 设置环境变量

```powershell
$env:TAURI_SIGNING_PRIVATE_KEY = (Get-Content "src-tauri/update-key.key" -Raw).Trim()
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD = "x"
```

> `Get-Content -Raw` 读取文件内容，`.Trim()` 去除尾随换行符，防止 base64 污染。
> 密码 `x` 替换为你生成密钥时设置的密码。

### 2.2 构建

```powershell
npm run tauri build
```

构建完成后产物位置：

```
src-tauri/target/release/bundle/
├── nsis/
│   ├── AI Canvas_0.2.7_x64-setup.exe      ← 安装包
│   └── AI Canvas_0.2.7_x64-setup.exe.sig  ← 签名文件
├── msi/
│   ├── AI Canvas_0.2.7_x64_en-US.msi
│   └── AI Canvas_0.2.7_x64_en-US.msi.sig
└── ...
```

### 2.3 检查产物

确认以下文件均已生成：
- `.exe` / `.msi` 安装包
- 对应的 `.sig` 签名文件
- 终端无报错

---

## 第三步：创建 `latest.json`

Tauri 更新器通过拉取 `latest.json` 来判断是否有新版本。本地构建不会自动生成此文件。

### 3.1 模板

```json
{
  "version": "0.2.7",
  "notes": "本版更新说明……",
  "pub_date": "2026-07-06T11:30:00Z",
  "platforms": {
    "windows-x86_64": {
      "signature": "<把 .sig 文件内容贴到这里>",
      "url": "https://github.com/Tenney95/AI-Canvas-tauri/releases/download/v0.2.7/AI.Canvas_0.2.7_x64-setup.exe"
    }
  }
}
```

- `signature`：复制 `AI Canvas_0.2.7_x64-setup.exe.sig` 文件里的全部内容
- `url`：GitHub Release 的直接下载链接。注意 GitHub 会把文件名中的空格替换为 `.`
- `pub_date`：当前 UTC 时间，格式 `YYYY-MM-DDTHH:mm:ssZ`

### 3.2 上传方式

**通过 CI 自动发布（推荐）**：跳过此步，`tauri-action` 的 `includeUpdaterJson: true` 会自动生成并上传。

**手动发布**：需自行创建 `latest.json` 并上传到 Release Assets。

---

## 第四步：发布 GitHub Release

### 方式 A：自动发布（推荐）

```powershell
git add -A
git commit -m "chore: release v0.2.7"
git tag v0.2.7
git push origin master --tags

# 强制覆盖 tag
git tag -f v0.4.2
git push origin -f v0.4.2
```

推送 tag 后，`.github/workflows/release.yml` 自动触发：
1. 构建 Windows / macOS / Linux 全平台安装包
2. 生成 `.sig` 签名文件
3. 生成 `latest.json`
4. 创建 GitHub Release 并上传所有 assets

### 方式 B：手动发布

1. 打开 https://github.com/Tenney95/AI-Canvas-tauri/releases/new
2. Tag 填 `v0.2.7`，Release title 填 `AI Canvas v0.2.7`
3. 上传以下文件到 Assets：
   - `AI Canvas_0.2.7_x64-setup.exe`（安装包）
   - `AI Canvas_0.2.7_x64-setup.exe.sig`（签名）
   - `latest.json`（版本描述）
4. 填写 Release notes，点击 **Publish release**

> 手动发布时必须上传 `latest.json`，否则自动更新会报 `Could not fetch a valid release JSON`。

---

## 第五步：验证更新

### 5.1 本地验证

1. 把版本号回退到上一版（如 `0.2.6`）
2. 重新构建安装包并安装
3. 启动应用 → 观察控制台 → 应自动拉取 `latest.json` 并开始下载新版本
4. 下载完成后应用自动重启至新版本

### 5.2 验证要点

- 控制台输出 `[Updater] 发现新版本 v0.2.7，开始下载...`
- 进度条显示下载进度
- 下载完成后自动安装并重启
- 重启后版本号显示为新版本
- 如果在已是最新版的机器上打开，输出 `当前已是最新版本`

### 5.3 常见问题

| 现象 | 原因 | 解决 |
|---|---|---|
| `开发模式，跳过更新检查` | `npm run tauri dev` 运行 | 正常，打包后自动不跳过 |
| `Could not fetch a valid release JSON` | `latest.json` 未上传到 Release Assets | 手动上传 |
| `pubkey is not valid` / 签名验证失败 | 本地构建用的 key 与 GitHub Release 的签名不匹配 | 确保用同一密钥对 |
| `failed to decode secret key` | 环境变量未设置或 base64 含换行符 | `Get-Content -Raw` 后加 `.Trim()`，并设置密码 |
| `failed to decode base64 secret key` | `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` 未设置 | 检查密码是否与生成密钥时一致 |

## 一图流

```
修改 package.json version
        │
        ▼
npm run sync-version  ──→ 同步 Cargo.toml + README
        │
        ▼
手动改 tauri.conf.json version
        │
        ▼
$env:TAURI_SIGNING_PRIVATE_KEY = ...
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD = ...
npm run tauri build
        │
        ▼
确认产物: .exe + .sig
        │
        ├─ 自动发布 ──→ git tag v0.2.7 && git push --tags
        │
        └─ 手动发布 ──→ 创建 latest.json → 上传到 GitHub Release
```
