# ADR 0004：使用会话级本地 MCP 控制桥

## 状态

已接受，2026-07-24。

## 背景

AI Canvas 的录制和批量操作需要比 Windows 坐标自动化更稳定的控制面。项目已经具备 Tool Registry、Policy Engine、Agent Runtime、任务时间线、画布 revision、文件 grant 和媒体逐次审批，但没有标准的外部工具协议。直接让外部进程访问 Zustand、IndexedDB、任意 HTTP、Shell 或文件系统会绕过现有安全边界，也会破坏主窗口 Store 的唯一写入源约束。

目标是在应用明确开启的单次会话中，让 Codex 等 MCP 客户端发现和调用现有 Agent 工具，同时保留项目、会话、任务、审批、审计、撤销和模型选择语义。MCP 调用不能批准自己的高影响操作，不能读取 API Key，也不能扩大 Tool Registry 的能力集合。

## 决策

1. 使用官方 `@modelcontextprotocol/sdk` 实现 stdio MCP 适配器，避免自行维护 MCP 初始化、消息 framing、工具发现和协议兼容。
2. 适配器只连接 Tauri 在 `127.0.0.1:0` 创建的随机端口。应用每次手动开启会话时由 Web Crypto 生成 256 位一次性令牌；令牌只保存在前端组件状态、Rust 会话内存和适配器进程参数中。
3. 回环协议使用版本化、限长的单行 JSON。每个请求携带令牌；Rust 在把请求发送到 WebView 前移除令牌。无效令牌、超长帧、未知方法、超时和过期会话均返回固定错误，不写入业务状态。
4. Rust 仅承担端口、鉴权、会话生命周期、请求关联和 Tauri Event 转发。它不解释 Agent 工具、不访问 IndexedDB，也不执行画布、媒体、文件或配置操作。
5. 主窗口 MCP 控制服务是唯一请求处理者。工具发现来自现有 Tool Registry；调用继续经过本地 schema、可用性、Policy Engine、执行前项目与授权复核、只读重试、画布 checkpoint 和结果脱敏。
6. 每个项目按需创建标题为“`MCP 控制`”的专用对话。每次调用创建可持久化 AgentTask 和步骤，审批、失败、revision 与撤销 checkpoint 使用现有任务时间线展示。
7. MCP 会话默认关闭，只能在设置页手动开启。停止、窗口关闭或应用退出立即使令牌失效；不把端口、令牌或绝对本地路径写入配置、IndexedDB、Agent 摘要或日志。
8. MCP 不能调用审批解决接口。`file_write`、`permanent_delete`、`media_generation`、`memory_write`、`config_write` 以及 B 模式的 `canvas_write` 仍由 AI Canvas 内的审批卡片确认。
9. MCP 只暴露已注册的领域工具和脱敏应用状态。不增加通用 Shell、任意路径文件、任意 HTTP、直接 Store 写入或直接 Provider 调用。

## 非功能要求

- **安全性**：仅监听 IPv4 loopback；一次性 256 位令牌；令牌不进入 Tauri Event 和持久化数据；1 MiB 请求上限；固定方法白名单。
- **一致性**：所有业务写入仍从主窗口 Store 发起；项目 ID、会话 ID、task ID 和 canvas revision 在执行前复核。
- **可恢复性**：连接或适配器失败不影响应用；停止会话后待处理请求失败且不能在新会话重放。
- **可审计性**：每个 MCP 工具调用都有 AgentTask、Policy 决策、步骤结果和可选画布 checkpoint。
- **兼容性**：stdio 侧遵循官方 SDK；内部回环协议带版本字段，宿主与适配器版本不匹配时明确失败。
- **性能**：单次会话只接受一个活跃适配器连接；请求串行关联，高影响审批可长时间等待但不阻塞 UI 线程。

## 失败策略

- 端口绑定失败时保持关闭并在设置页显示错误。
- 认证失败后立即返回错误并关闭连接，不向前端发事件。
- 前端未就绪、请求超时或会话被停止时，适配器收到结构化错误；写工具不自动重试。
- 应用切换项目后，尚未执行的调用在执行前复核失败；不会写入新项目。
- 用户拒绝或停止审批后，任务记录为跳过、停止或失败；MCP 收到对应错误文本，不能再次提交同一审批。
- stdio 客户端退出不会自动重新开启会话；用户可在设置页停止后重新生成令牌。

## 未采用方案

- **Windows 坐标自动化**：窗口大小、焦点、缩放和动画会导致不稳定，且难以建立审计和撤销边界。
- **前端直接启动 stdio MCP**：WebView 不应持有子进程和标准输入输出能力，也会扩大 Tauri capability。
- **固定端口或持久令牌**：方便配置但扩大本机驻留攻击面，不符合手动会话授权。
- **外部进程直连 IndexedDB/Store**：绕过项目隔离、历史、Policy 和主窗口单写源。
- **MCP 自带审批工具**：客户端可以自批高影响操作，破坏固定安全矩阵。
- **手写 MCP 协议栈**：短期减少依赖，但初始化、兼容、取消和 framing 的长期维护风险更高。

## 影响

- 每次重新开启会话都需要更新客户端使用的端口和令牌。
- AI Canvas 必须保持运行且主窗口已初始化，外部 MCP 工具才可用。
- 受保护调用可能在 MCP 客户端侧等待用户回到应用审批。
- 新增一个 npm 运行时依赖；Rust 只扩展现有 Tokio 的网络与 I/O feature，不新增 Cargo crate。

## 回滚

先在设置页停止 MCP 会话，再移除 stdio 适配器、前端 MCP 服务和 Rust `mcp_bridge` 模块，恢复 `lib.rs`、`Cargo.toml`、设置导航与 Agent 工具执行器调用点。回滚不涉及数据库迁移、配置迁移、项目数据或 Tauri 安全配置。
