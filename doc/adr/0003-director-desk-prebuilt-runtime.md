# ADR 0003：3D 导演台使用按需下载运行时

## 状态

已接受，2026-07-21。取代本 ADR 早期的“构建时下载并随安装包分发”方案。

## 背景

3D 导演台通过固定标签 `director-desk` 的 Tauri 独立窗口接入主项目，并使用带 `instanceId`、`requestId` 的 Tauri Event 协议通信。应用运行不应依赖导演台源码、Git、Node.js、npm 或本地端口；需要开发导演台本体时，应直接在其独立仓库启动开发服务器。

构建时下载预构建资源虽然解决了正式环境启动开发服务器的问题，但仍会让每个 AI Canvas 安装包增加约 82 MB，即使用户从不创建导演台节点也要承担下载和磁盘成本。因此正式环境改为首次使用时按需安装固定发布资产。

## 决策

1. `Tenney95/3d-director-desk` 继续通过 Git tag 发布版本化 `tar.gz` 静态资源包，并提供版本、协议元数据和 SHA-256。
2. 主项目在 `scripts/director-desk-release.json` 固定仓库、版本、产物 URL、协议、大小和 SHA-256；运行时不跟随分支或 `latest`。
3. 用户新建导演台节点或打开缺少运行资源的导演台时，由主窗口显示一次下载确认。所有导演台节点共用同一份资源和单一安装任务。
4. Rust 后端将资源流式下载到 `appLocalDataDir/director-desk`，同步计算 SHA-256，并限制下载大小、展开大小、文件数量、归档类型和路径。
5. 资源先解压到 staging 目录，校验 `index.html`、版本和协议元数据后再切换到版本目录。取消或失败会删除临时文件，不破坏已安装版本。
6. 开发与正式环境统一通过 `director-desk://localhost/index.html` 加载资源，不启动 HTTP 服务。协议处理器只允许标签为 `director-desk` 的窗口访问固定版本目录，并实施规范路径包含校验、MIME 和单段 Range 响应。
7. `npm run dev` 和 `npm run tauri dev` 不再克隆源码、安装依赖或自动启动 5178；Tauri 开发环境首次使用时执行与正式环境相同的下载、校验和安装流程。
8. 设置的“存储健康”页提供安装状态、版本、占用和删除入口。Windows NSIS 卸载前只删除 `$LOCALAPPDATA/com.aicanvas.app/director-desk`；macOS 和 Linux 由应用内入口清理。

## 非功能要求

- **按需体积**：未使用导演台的用户不会下载其静态资源，资源也不进入 AI Canvas 的 Vite 或安装包产物。
- **环境一致性**：开发与正式环境使用同一固定产物和加载协议，避免只在开发服务器中成立的行为。
- **离线复用**：当前固定版本安装并校验成功后，后续打开直接读取本地资源，无需网络和端口。
- **可复现性**：每个 AI Canvas 版本固定一个导演台版本和 SHA-256；校验失败必须中止安装，禁止静默切换版本。
- **安全性**：不执行下载内容中的脚本进程，不暴露任意文件或通用 HTTP 能力，不放宽现有 Tauri capability。
- **可恢复性**：安装过程使用临时文件和 staging 目录；取消、网络失败或解压失败后可以重试。
- **数据边界**：卸载和手动清理只删除导演台运行资源，不删除用户项目、配置、对话或其他应用数据。

## 失败策略

- 当前版本已安装时直接使用，不访问网络。
- 下载、哈希、磁盘空间、归档安全或发布元数据校验任一失败时保留节点，显示错误并允许重试。
- 用户取消时保留节点并清理本次临时内容，下次打开继续显示同一下载确认。
- 升级导演台必须显式更新固定清单并完成协议兼容验证；旧版本最多保留一份用于安装期恢复，后续自动清理。

## 未采用方案

- **生产环境拉源码并启动 Vite**：依赖开发工具链、网络和端口，启动慢且不可复现。
- **默认开发流程自动启动导演台 Vite**：使开发验证绕过真实按需安装链路，并额外占用终端和端口；导演台源码调试应在 Fork 仓库独立进行。
- **随主安装包捆绑预构建资源**：运行可靠，但所有用户都承担约 82 MB 体积，不符合按需目标。
- **生产环境启动 Rust 静态文件服务器**：引入端口生命周期、占用和劫持面，自定义本地协议已经能满足静态资源加载。
- **直接跟随 GitHub `latest`**：无法保证版本和宿主协议兼容，禁止使用。
- **下载到应用安装目录**：普通用户通常无写权限，升级和卸载行为也不稳定；使用系统定义的应用本地数据目录。

## 影响

- 首次使用导演台需要下载约 54 MB，安装后约占 82 MB。
- AI Canvas 安装包不再包含导演台资源，生产构建无需访问导演台 Release。
- Windows 卸载会自动清理下载内容；其他平台可在应用内提前删除，系统卸载器对应用数据目录的处理仍由平台决定。

## 回滚

前端可恢复构建期资源路径和准备脚本，Rust 命令与协议注册可以独立移除。回滚发布时应恢复整个兼容版本方案，不得只替换运行资源并绕过固定清单和协议校验。
