# ADR 0003：3D 导演台使用版本化预构建资源

## 状态

已接受，2026-07-21。

## 背景

3D 导演台已经通过固定标签 `director-desk` 的 Tauri 独立窗口接入主项目，并使用带 `instanceId`、`requestId` 的 Tauri Event 协议通信。原实现无论开发还是生产都依赖 `127.0.0.1:5178`，启动包装脚本还会在本机克隆源码、安装 npm 依赖并启动 Vite。这要求最终用户具备 Git、Node.js、npm 和可用端口，也把运行稳定性绑定到网络、源码分支与开发服务器，不适合作为正式安装包的运行方式。

## 决策

1. `Tenney95/3d-director-desk` 以 Git tag 发布版本化的 `dist` 静态资源包，发布前运行完整测试与生产构建，同时提供 SHA-256 校验文件。
2. 主项目通过 `scripts/director-desk-release.json` 固定仓库、版本、产物 URL、协议版本和 SHA-256。生产构建前由 `prepare-director-desk.mjs` 下载并验证指定产物，不跟随 `main` 或 `latest`。
3. 准备脚本限制包体大小，拒绝绝对路径和 `..` 路径，验证发布元数据后优先原子替换 `public/director-desk`。Windows 文件监听器持续占用目录时，先保留旧版本备份，再退化为已校验目录复制。相同版本可复用已验证缓存。
4. Vite 将 `public/director-desk` 一并写入主应用前端产物。正式安装后，独立窗口加载 `director-desk/index.html`，不启动 HTTP 服务，也不访问网络或执行 Git、Node.js、npm。
5. 开发环境保持原有体验：`npm run dev` 或 `npm run tauri dev` 可从本地源码启动 Vite，并加载 `http://127.0.0.1:5178`。本地 origin 配置只在开发模式生效。
6. 通信协议、固定窗口标签、节点 `instanceId`、请求 `requestId` 和主窗口唯一写入源均保持不变。现有 capability 继续只授予 Event listen/unlisten/emit；开发远程 URL 仍限定为 `127.0.0.1:5178`，生产本地页面不新增权限。

## 非功能要求

- **启动与离线**：安装后的导演台直接读取应用资源，可离线打开，不受端口占用影响。
- **可复现性**：每个 AI Canvas 版本固定一个导演台版本和 SHA-256，构建时校验失败必须终止，不静默切换到远程最新版本。
- **安全性**：产物只接受 HTTPS 固定发布地址，下载大小受限，解包前检查路径，运行窗口不获得文件、Shell、Dialog 或通用网络权限。
- **可恢复性**：构建缓存保留上一份资源；正式环境按 AI Canvas 安装包整体回滚，避免主应用与导演台协议版本错配。
- **兼容性**：预构建资源继续使用相对资源路径 `base: "./"`，支持 Windows、macOS 和 Linux 的 Tauri 本地资源加载。

## 失败策略

- 网络不可用但当前版本已通过校验并缓存时，继续使用缓存。
- 干净构建机无法下载指定版本、SHA-256 不匹配、包体超限、归档路径不安全或版本/协议元数据不匹配时，立即中止构建。
- 不允许构建过程自动退回旧导演台并生成新版本安装包。需要回滚时恢复上一版发布清单，或回滚整个 AI Canvas 版本后重新构建。

## 未采用方案

- **生产环境拉源码并启动 Vite**：依赖开发工具链、网络和端口，启动慢且不可复现。
- **生产环境启动 Rust 静态文件服务器**：仍引入端口生命周期、占用与劫持面；Tauri 已能加载应用内静态资源，没有必要增加服务进程。
- **运行时下载并热更新导演台**：会产生主协议与子应用版本漂移，也增加更新失败后的恢复复杂度。本阶段随主应用版本一起发布。
- **直接跟随 GitHub `latest`**：无法保证可重复构建和兼容性，禁止使用。
- **提交导演台 `dist` 到主仓库**：增加大体积构建产物和无意义差异，改用带校验的发布资产。

## 影响

- 正式构建首次需要访问固定 GitHub Release；已验证缓存可用于后续离线重建。
- AI Canvas 升级导演台时必须显式更新发布清单并通过协议兼容验证。
- 最终安装包会增加导演台压缩前静态资源体积，但不再携带其源码、依赖目录或开发服务器。

## 回滚

将 `scripts/director-desk-release.json` 恢复到上一兼容版本及其 SHA-256，重新构建并发布 AI Canvas；已安装客户端可通过上一版完整安装包回滚。不得只替换运行时子资源而绕过主应用版本管理。
