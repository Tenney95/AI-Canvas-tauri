# AI Canvas 架构说明

> 本文档描述 AI Canvas 项目的整体架构设计、分层职责、数据流和关键设计决策。
> 最后更新: 2026-06-05

## 1. 项目概述

AI Canvas 是一个基于 **Tauri 2 + React Flow** 的桌面端 AI 多模态节点画布编辑器。用户通过可视化节点和连线，组合 AI 文本生成、图像生成、视频生成、音频生成等能力，构建复杂的 AI 工作流。

## 2. 分层架构

### 2.2 状态管理层 (Store)

采用 Zustand 5 的 **Slice 模式**，将全局状态按职责拆分为 10 个独立 slice：

| Slice | 文件 | 管理状态 |
|-------|------|---------|
| nodes | `store.nodes.ts` | 画布节点/边 CRUD、选中状态 |
| ui | `store.ui.ts` | 面板开关、菜单位置、弹窗状态 |
| toast | `store.toast.ts` | 全局消息提示队列 |
| history | `store.history.ts` | 撤销/重做历史栈（最多 50 条） |
| config | `store.config.ts` | API 提供商配置（Key/URL） |
| workflows | `store.workflows.ts` | ComfyUI 工作流管理 |
| presets | `store.presets.ts` | 用户自定义预设提示词 |
| groups | `store.groups.ts` | 节点分组 |
| clipboard | `store.clipboard.ts` | 复制/粘贴（含外部剪贴板内容） |
| projects | `store.projects.ts` | 多项目管理、保存/加载/切换 |

所有 slice 通过 `src/store/useAppStore.ts` 聚合为一个 Zustand store：

```ts
export const useAppStore = create<AppState>()((...a) => ({
  ...createNodeSlice(...a),
  ...createUISlice(...a),
  // ... 其余 8 个 slice
}));
```

**状态流规则**：
- 组件只读状态通过 `useAppStore(selector)` 获取
- 所有变更必须通过 Store Actions
- 组件内禁止直接修改 store 状态
- 新增状态优先归入对应 slice，不散落在组件中

## 3. 数据流

### 3.1 节点创建流程

```
用户操作 (Sidebar/右键菜单/双击)
  → useAppStore.addNode()
    → commitToHistory() 保存快照
    → nodes 数组追加新节点
    → React Flow 重新渲染
```

### 3.2 AI 生成流程

```
用户在 TextNode/ImageNode/VideoNode/AudioNode 点击「生成」
  → aiService 根据 provider + model 选择对应适配器
  → 构建请求（文本：Chat Completions / 图像：Image Generations / 视频/音频：Task Submit）
  → 对于异步任务（图像/视频/音频），轮询任务状态
  → 结果回填到节点 data (output / imageUrl / videoUrl / audioUrl)
  → 更新节点状态 (idle → loading → success/error)
```

### 3.3 持久化流程

```
用户 Ctrl+S / 切换项目 / 应用退出
  → saveCurrentProject()
    → fileService.saveProject() → indexedDbService.saveProjectToDb()
    → 节点/边/分组数据序列化存入 IndexedDB

应用启动
  → App.initFromDb()
    → loadConfig() + loadWorkflows() + loadPresets()
    → loadProjectsList() → 恢复上次打开的项目
```

## 5. 关键设计决策

1. **Zustand Slice 模式**：避免单一巨型 store，按职责拆分 10 个 slice，便于维护和测试
2. **IndexedDB 持久化**：浏览器端首选持久化方案，同时支持 Tauri 原生文件系统作为文件存储补充
3. **Tauri + Web 双环境兼容**：`fileService` 通过 `isTauriEnv()` 判断运行环境，自动降级到浏览器 API
4. **模型配置化**：通过 `defaultModels.ts` 预置模型列表，`ModelSelector` 组件统一渲染，避免硬编码分支
5. **@ 引用语法**：`MentionEditor` 组件实现跨节点引用，支持 `@节点名` 语法将其他节点输出注入当前提示词
6. **历史栈撤销/重做**：`store.history.ts` 维护最多 50 条快照，每次变更前 `commitToHistory()` 保存状态
7. **自定义窗口装饰**：Tauri `decorations: false` + `transparent: true`，前端用 `Titlebar.tsx` 实现自定义标题栏
