# AI Canvas 架构说明

> 本文档描述 AI Canvas 当前的系统边界、模块职责、核心数据流和扩展约束。
> 当前版本：`0.6.0`；最后更新：2026-07-21。

## 1. 项目定位

AI Canvas 是一个基于 **Tauri 2 + React 19 + React Flow 12** 的桌面端 AI 多模态画布。你可以在无限画布中组织文本、图像、视频、音频、动画、全景图、Markdown、分镜和 3D 导演台节点，通过连线、引用、批量执行及对话 Agent 完成内容生成与画布操作。

项目同时支持：

- Tauri 桌面环境与有限的浏览器降级运行；
- 多项目、工作流、模型预设、技能和风格管理；
- 多厂商文本、图像、视频、音频生成；
- ComfyUI 工作流、Dreamina 登录态调用和本地 ONNX 推理；
- 主画布、独立聊天窗口、独立资产搜索窗口和 3D 导演台窗口。

## 2. 总体架构

```text
┌──────────────────────────────────────────────────────────────┐
│ Tauri 2 桌面层                                               │
│ Rust commands / plugins / window / fs / updater / ONNX       │
│ Director Desk download / validation / local URI protocol     │
└──────────────────────────────┬───────────────────────────────┘
                               │ IPC
┌──────────────────────────────▼───────────────────────────────┐
│ React 19 表现层                                               │
│ App / Canvas / Nodes / Panels / Agent / Independent Windows  │
└──────────────────────────────┬───────────────────────────────┘
                               │ actions + selectors
┌──────────────────────────────▼───────────────────────────────┐
│ Zustand 5 全局状态                                            │
│ 18 slices：画布、项目、聊天、Agent、记忆、配置和 UI 等          │
└──────────────────────────────┬───────────────────────────────┘
                               │ service calls
┌──────────────────────────────▼───────────────────────────────┐
│ 领域服务与运行时                                               │
│ AI generation / chat pipeline / polling / batch DAG / files  │
└───────────────┬──────────────────────────────┬───────────────┘
                │                              │
┌───────────────▼──────────────┐ ┌────────────▼────────────────┐
│ IndexedDB / 项目数据目录      │ │ 外部模型 API / ComfyUI       │
│ appLocalDataDir 运行资源      │ │ Dreamina / 本地 ONNX         │
└──────────────────────────────┘ └─────────────────────────────┘
```

核心边界如下：

- 组件负责展示和收集交互，不直接实现厂商协议或持久化细节；
- `useAppStore` 是前端共享业务状态的唯一入口；
- `services/` 负责 AI 请求、轮询、文件、数据库和聊天管线；
- Rust 层负责浏览器无法安全或稳定完成的系统能力；
- 生成结果优先保存到项目数据目录，Store 中保存可恢复的资源引用。
- 外部可选运行时必须锁定版本和校验值，并安装到应用本地数据目录。

## 3. 运行入口与窗口

`src/main.tsx` 复用同一套前端入口，通过 URL 参数选择 AI Canvas 根视图：

| 入口 | 根组件 | 用途 |
|---|---|---|
| 无 | `App` | 主画布窗口 |
| `?view=chat` | `ChatWindow` | 独立聊天助手窗口 |
| `?view=assets` | `AssetSearchWindow` | 独立资产搜索窗口 |
| `director-desk://localhost/index.html` | 预构建 3D 导演台 | 固定标签 `director-desk` 的独立窗口 |

`App.tsx` 负责装配主窗口级组件，包括 `Canvas`、`Header`、`Sidebar`、`Titlebar`、节点菜单、设置、工作流、资产、输出历史、聊天面板、Toast、开屏动画、吉祥物和更新提示。应用启动时从 IndexedDB 恢复数据，并在 Tauri 窗口关闭前静默保存当前项目和清理撤销暂存目录。

3D 导演台不经过 `src/main.tsx`，也不嵌入 iframe。`directorDeskWindowService.ts` 创建或复用单一 `WebviewWindow`，使用 `instanceId` 将当前导演台节点绑定到独立窗口。主窗口是节点数据的唯一写入源。

## 4. 前端表现层

### 4.1 画布与交互

`src/components/Canvas.tsx` 是 React Flow 的集成边界，负责节点/边渲染和画布事件编排。复杂交互拆分到 `components/canvas/` 与 `hooks/`：

| 模块 | 职责 |
|---|---|
| `CanvasToolbar` | 缩放、视图和画布工具 |
| `CanvasContextMenu` / `NodeContextMenu` | 画布与节点右键操作 |
| `ConnectionMenu` | 连线释放后的节点选择 |
| `MultiSelectToolbar` | 多选节点对齐、分布和批量操作 |
| `DistributionGapHandles` | 平均分布后的间距拖拽调整 |
| `useNodeCreation` | 节点创建和文件拖入 |
| `useNodeSnap` | 单选与多选包围盒吸附、对齐参考线 |
| `useKeyboardShortcuts` | 保存、撤销、删除，以及数字键 `1-7` 创建内容节点等全局快捷键 |
| `useAutoSave` | 项目自动保存 |

### 4.2 节点体系

节点类型统一定义在 `src/types/index.ts`。当前渲染组件包括：

| 组件 | 主要职责 |
|---|---|
| `TextNode` | 文本提示词、引用、流式文本生成与编辑 |
| `ImageNode` | 文生图/图生图、上传、裁剪、扩图、标注、抠图等 |
| `VideoNode` | 文生视频/图生视频与视频参数 |
| `AudioNode` | 音频生成、上传和播放 |
| `PanoramaNode` | 全景图生成与预览 |
| `AnimationNode` | 逐帧动画参数、生成与播放预览 |
| `DirectorDeskNode` | 导演台独立窗口、截图同步和下游媒体引用 |
| `MarkdownNode` | Markdown 内容编辑与渲染 |
| `StoryboardNode` | 分镜宫格、单格覆盖与素材组织 |
| `GroupNode` | 节点分组容器 |

共享编辑器和选择器位于 `components/nodes/shared/`，其中包括模型选择、提示词 Composer、@引用、预设、技能、风格、质量比例，以及裁剪、扩图、自由角度、标注、抠图和宫格编辑等能力。新增通用节点交互应优先复用这里的组件。

`components/director/` 管理导演台运行资源的状态检查、下载确认、进度和错误恢复。`components/settings/DirectorDeskStorageManager.tsx` 展示已安装版本与占用，并提供应用内删除入口。

### 4.3 聊天助手

`components/chat/` 提供嵌入式和独立窗口共用的聊天 UI。业务逻辑由以下模块协作：

- `agentRuntime.ts`：多轮“模型、工具、Observation、模型”循环、任务控制、预算和审批等待；
- `toolRegistry.ts`：工具注册、可用性过滤和本地 schema 校验；
- `policyEngine.ts`：B/C 模式与工具 effect 的固定权限矩阵；
- `agentTaskService.ts` / `store.agent.ts`：任务持久化、后台状态和重启修复；
- `contextManager.ts` / `contextCompressionService.ts`：上下文预算、历史组装和压缩；
- `projectMemoryService.ts` / `store.memory.ts`：用户确认的项目记忆；
- `fileGrantService.ts`：绑定会话的临时本地文件授权；
- `services/ai/assistantStream.ts`：Provider 流式文本和工具调用协议。

B 模式的画布写操作需要确认，C 模式可自动执行。文件写入、永久删除、媒体生成和记忆写入始终需要确认。画布写入还必须校验 `projectId`、canvas revision，并提交撤销历史。

## 5. 状态管理

`src/store/useAppStore.ts` 使用 Zustand Slice 模式聚合 18 个 slice：

| Slice | 职责 |
|---|---|
| `store.nodes.ts` | 节点、边、选择状态及节点数据更新 |
| `store.ui.ts` | 面板、菜单、弹窗和画布 UI 状态 |
| `store.toast.ts` | 全局通知 |
| `store.history.ts` | 画布快照的撤销/重做 |
| `store.historyRecord.ts` | 节点生成输出历史记录 |
| `store.config.ts` | 主题、模型提供商、通用模型及应用配置 |
| `store.workflows.ts` | ComfyUI 工作流 |
| `store.presets.ts` | 提示词预设 |
| `store.skills.ts` | 技能库 |
| `store.styles.ts` | 风格库 |
| `store.groups.ts` | 节点分组 |
| `store.clipboard.ts` | 节点及系统剪贴板操作 |
| `store.projects.ts` | 多项目、保存、加载和切换 |
| `store.chat.ts` | 聊天会话、消息、请求和画布操作日志 |
| `store.agent.ts` | AgentTask、步骤、审批和后台任务状态 |
| `store.memory.ts` | 项目记忆及候选确认状态 |
| `store.toolbar.ts` | 节点工具栏布局 |
| `store.dramaAssets.ts` | 短剧人物、场景和道具资产 |

状态使用规则：

1. 组件通过 selector 订阅所需的最小状态；
2. 共享业务状态必须通过 Store Action 修改；
3. 异步协议和持久化细节留在 Service，slice 只负责编排和状态收口；
4. 画布结构性变更应接入撤销历史和聊天 revision；
5. 大体积媒体数据不直接写入 Store，保存文件路径、asset URL 或远端源地址。

## 6. 服务层与 AI 运行时

### 6.1 AI 生成

`services/aiService.ts` 保留统一公开入口，具体实现拆分在 `services/ai/`：

- `generateText.ts`、`generateImage.ts`、`generateVideo.ts`、`generateAudio.ts`：按媒体类型组织请求；
- `providers/`：厂商特定适配器；
- `promptResolver.ts`、`nodeReferenceService.ts`：解析节点引用和生成输入；
- `pollTask.ts`、`pollManager.ts`：异步生成任务轮询与生命周期管理；
- `comfyWorkflowService.ts`：ComfyUI 工作流执行；
- `dreaminaService.ts`：Dreamina 登录态和生成调用；
- `onnxService.ts`：本地超分和主体抠图接口；
- `uploadService.ts`：需要公网输入时的媒体上传与 URL 转换。

新增厂商差异应收口到适配器或对应 service，避免在节点组件和 Store 中散落 provider 分支。

### 6.2 批量执行

`src/utils/batchExecute.ts` 负责选中节点的批量生成：

1. 过滤缺少模型、提供商、提示词或正在执行的节点；
2. 对选中范围内有连线的节点使用 Kahn 算法拓扑排序并顺序执行；
3. 无依赖的孤立节点并发执行；
4. 两组任务并行推进；
5. 每个结果更新节点状态、保存媒体并写入输出历史。

### 6.3 文件与持久化

| 服务 | 职责 |
|---|---|
| `fileService.ts` | 项目文件、生成结果、导入导出和跨环境文件操作入口 |
| `services/fs/core.ts` | 文件系统基础能力 |
| `services/fs/assetLibrary.ts` | 项目资产库 |
| `services/fs/trash.ts` | 可撤销删除与清理 |
| `indexedDbService.ts` | 浏览器侧结构化持久化 |
| `storageService.ts` | 轻量本地存储封装 |
| `updateService.ts` | 桌面应用更新检查与安装 |

### 6.4 3D 导演台运行时

3D 导演台来自独立仓库 `Tenney95/3d-director-desk`。主项目不拉取源码、不执行 npm 安装，也不启动额外 HTTP 服务。开发与生产环境使用同一条运行链路：

1. `scripts/director-desk-release.json` 固定仓库、版本、产物 URL、协议、大小和 SHA-256；
2. 创建或打开 `ai-director` 节点时，Store 请求 `DirectorDeskRuntimeManager` 检查运行资源；
3. 用户确认后，前端调用 Rust command 下载、校验并安装资源；
4. `directorDeskWindowService.ts` 通过 `director-desk://localhost/index.html` 打开独立窗口；
5. 主窗口与导演台使用限定消息类型的 Tauri Event 信封通信；
6. 截图和扩展操作结果回写当前 `instanceId` 对应的画布节点。

运行资源安装在 `appLocalDataDir/director-desk/<version>`。安装过程使用临时下载文件和 staging 目录，失败或取消时清理临时内容。成功安装后保留当前版本和最多一个旧版本。

应用内删除入口会关闭导演台窗口并移除整个运行资源目录。Windows NSIS 卸载钩子也会删除该目录；macOS 和 Linux 依赖应用内入口或平台对应用数据目录的清理策略。项目、配置和对话数据不属于该清理范围。

## 7. 数据持久化

IndexedDB 数据库名为 `ai-canvas-db`，当前 schema 版本为 13，包含：

| Object Store | 内容 |
|---|---|
| `projects` | 项目元数据、节点和边 |
| `workflows` | ComfyUI 工作流定义 |
| `config` | 应用及模型配置 |
| `presets` | 提示词预设 |
| `history` | 节点输出历史 |
| `assetMeta` | 本地资产元数据 |
| `assetMetaV2` / `assetIndex` | 资产标识、路径、指纹和项目索引 |
| `styles` | 风格库 |
| `skills` | 技能库 |
| `chatConversations` | 聊天会话、置顶及软删除信息 |
| `chatMessages` | 会话消息、请求状态、命令和媒体结果 |
| `agentTasks` | Agent 任务、步骤、审批和恢复状态 |
| `projectMemories` | 用户确认的项目记忆及来源 |
| `toolbarLayouts` | 节点工具栏布局配置 |

项目媒体文件由 `fileService` 保存到项目数据目录，数据库和项目记录只保存元数据及引用。浏览器环境无法使用的原生能力应显式降级，不应伪造成功结果。

## 8. Rust / Tauri 原生层

`src-tauri/src/lib.rs` 负责插件注册、窗口管理和 IPC command 装配。当前插件包括：

- `fs`、`dialog`、`global-shortcut`、`shell`；
- `drag`；
- `updater`、`process`。

原生能力主要分为：

| 类别 | 能力 |
|---|---|
| 网络与文件 | 代理请求、图片转 data URL、移入回收站、指定应用打开文件 |
| 窗口 | 开关独立聊天窗口、开发者工具、自定义窗口行为 |
| 3D 导演台 | 固定 Release 下载、校验、取消、删除和 `director-desk://` 文件响应 |
| Dreamina | 登录窗口、会话运行时、状态、退出、生成和结果查询 |
| ComfyUI | 启动本地 ComfyUI |
| ONNX | 模型目录、模型检查/下载、图片超分、主体抠图、GPU 状态 |
| 更新 | 检查、下载、安装并重启应用 |

Rust 子模块位于 `src-tauri/src/comfyui/`、`dreamina.rs`、`director_desk_runtime.rs`、`file_transfer.rs` 和 `onnx/`。Windows 下额外处理 WebView2 GPU 参数，并使用 ONNX Runtime DirectML 执行本地推理。

`director_desk_runtime.rs` 只接受固定 GitHub Release URL，并限制压缩包为 100 MB、展开内容为 300 MB、归档条目为 10,000 个。协议处理器只允许标签为 `director-desk` 的窗口访问当前版本目录，并验证规范路径、MIME 类型和单段 Range 请求。

## 9. 核心数据流

### 9.1 节点生成

```text
节点交互
  → Store 将节点置为 loading
  → 解析 @节点引用 / 技能 / 风格 / 模型参数
  → aiService 媒体入口
  → provider、ComfyUI、Dreamina 或 ONNX
  → 同步返回或 pollManager 轮询
  → fileService 保存结果
  → Store 回填 URL、路径、尺寸、状态
  → historyRecord 记录生成历史
```

### 9.2 聊天助手

```text
ChatInput
  → 本地规则引擎（高置信命令快速路径）
  → 未命中时调用 LLM 流式助手
  → 校验命令 / 工具调用 schema
  → canvasPlanner + commandRegistry 执行或请求确认
  → 更新画布 revision 与操作日志
  → 消息、命令结果和媒体状态写入 IndexedDB
```

### 9.3 项目恢复

```text
应用启动
  → initFromDb
  → 恢复配置、项目、工作流、预设、技能和风格
  → 加载当前项目节点/边
  → 迁移并加载输出历史
  → 加载项目聊天会话并修复异常中断状态
```

### 9.4 3D 导演台首次使用

```text
创建或打开 ai-director 节点
  → Store 检查当前固定版本是否安装
  → 主窗口显示下载确认
  → Rust 流式下载并计算 SHA-256
  → 校验归档路径、类型、体积、index.html 和协议元数据
  → staging 目录切换为版本目录
  → director-desk:// 加载本地静态资源
  → Tauri Event 绑定 instanceId 并交换会话、截图和请求结果
```

## 10. 样式与性能

- Tailwind CSS 3 用于组件级工具类，颜色复用 `canvas-*` token；
- 全局入口为 `src/index.css`，功能样式拆分到 `src/styles/*.css`；
- React Flow 覆盖集中在 `src/styles/reactflow.css`；
- 节点按媒体类型拆分样式，编辑器也按能力拆分；
- 节点组件使用 `React.memo`，Store 使用细粒度 selector；
- Three.js、GSAP 等大体积吉祥物依赖按需懒加载；
- `DirectorDeskNode` 和下载管理 UI 按需加载，导演台静态资源不进入主应用 `dist` 或安装包；
- 高频拖拽、缩放和对齐计算避免写入持久化层，交互结束后再提交历史。

## 12. 设计约束与演进方向

1. **Store 是状态入口，不是协议实现层**：厂商请求、文件和数据库逻辑继续留在 Service。
2. **统一生成入口**：节点、批量执行和聊天媒体生成应复用同一媒体实现，避免行为分叉。
3. **配置驱动模型**：模型能力、参数和 provider 映射应由配置与适配器描述。
4. **工作流与模型 API 分层**：ComfyUI 工作流协议不与普通厂商模型请求强行混用。
